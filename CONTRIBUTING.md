# Contributing to n8n_prod

Thank you for considering contributing to the n8n_prod project! Here are our guidelines to ensure a smooth collaboration:

## Production Workflow Guidelines
1. **Branching**: Always create a new branch for your feature or fix. Base it off the `main` branch and ensure to name it descriptively.
2. **Testing**: Ensure all tests pass before you make a pull request. Add any new tests to cover your added functionality.
3. **Documentation**: Update the documentation if your changes impact the user experience or functionality. Include relevant details in your pull request.

## Code Review Standards
- **Code Quality**: Write clean, maintainable code. Follow the existing coding conventions used in the project.
- **Commits**: Make sure your commits are atomic and meaningful. Use clear messages to describe what each commit does.
- **Review Process**: Be open to feedback. Reviewers will provide comments and suggestions; address these promptly.

## Deployment Procedures
1. **Merging**: Once your pull request is approved, it can be merged into the main branch. Ensure it passes all CI checks before merging.
2. **Tagging Releases**: After merging significant changes, tag the commit with a version number according to semantic versioning (e.g., v1.0.0).
3. **Notify Team**: After deployment, notify the team of the new changes and any relevant highlights for the release.

Thank you for your contributions! Together, we can make n8n_prod even better!
